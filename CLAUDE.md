## Architecture

### Feature-Based Structure

Code is organized under `src/features/<domain>/` with each feature owning its screens, API, hooks, store, and types. Shared infrastructure lives at the top level under `src/components/`, `src/lib/`, and `src/utils/`.

```
src/
├── features/<domain>/
│   ├── components/    # Feature-specific components (if needed)
│   ├── screens/
│   ├── api/           # Axios functions + queryKeys
│   ├── hooks/         # useQuery / useMutation wrappers
│   ├── store/         # Zustand slice for feature-local state (if needed)
│   └── types.ts
├── store/             # Global Zustand stores + selector hooks
│   └── useUserStore.ts
├── components/
│   ├── shared/        # ErrorBoundary, ScreenHeader, ToastProvider, StatusBar
│   └── ui/            # Button, Card, Text, Input, Icon, ScreenWrapper, Section, StateView
├── lib/
│   ├── apiClient.ts   # Axios instance + auth interceptor
│   └── queryClient.ts # React Query client configuration
├── utils/
│   ├── cn.ts          # clsx + tailwind-merge helper
│   └── validateEmail.ts
├── navigation/        # RootNavigator, MainTabsNavigator, types, hooks
├── constants/
│   └── variable.ts    # TAB_BAR_HEIGHT, getTabBarHeight()
└── types/
    ├── declarations.d.ts
    └── uniwind.d.ts   # Auto-generated by metro — do not edit
```

### Navigation

Two-level stack/tabs setup:

- `RootNavigator` — native stack, no header, slide-from-right animation
- `MainTabsNavigator` — bottom tabs (Home, Profile), custom height via `getTabBarHeight()`

Navigation types are in `src/navigation/types.ts`. Always use composite screen prop types (`CompositeScreenProps`) for screens nested inside tabs. Use `useRootNavigation()` from `src/navigation/hooks.ts` inside shared components that need navigation.

This project imports directly from `@react-navigation/*` — **not** from `expo-router`. Do not migrate imports to expo-router.

### API Layer

All API calls go through the shared Axios instance at `src/lib/apiClient.ts`:
- Base URL: `https://jsonplaceholder.typicode.com` (replace per project)
- Timeout: 10s
- Auth token automatically injected from AsyncStorage via request interceptor registered once in `useApiClient()` hook

React Query client is configured in `src/lib/queryClient.ts`.

Pattern for a new feature API:
1. Create `src/features/<domain>/api/<domain>.api.ts` — pure functions that accept `axios` as first arg
2. Create `src/features/<domain>/api/queryKeys.ts` — typed const arrays
3. Create hooks in `src/features/<domain>/hooks/` wrapping `useQuery` / `useMutation`; call `useApiClient()` inside

> **Note:** `apiClient` is a stable Axios instance — it is intentionally **not** included in `queryKey`. The `@tanstack/query/exhaustive-deps` rule is disabled globally for this reason.

### State Management

- **Server state**: React Query (`@tanstack/react-query`) — default `retry: 1`, `staleTime: 5min`
- **Global client state**: Zustand — lives in `src/store/use<Domain>Store.ts`; co-locate selector hooks (`useCurrentUser`, `useUserActions`) in the same file to avoid direct store access in components
- **Feature-local state**: Zustand — lives in `src/features/<domain>/store/use<Domain>Store.ts` for state that only one feature needs

### Styling System

Uses **Tailwind CSS v4 + Uniwind** (not NativeWind v4). The setup is:

- `global.css` — imports Tailwind and Uniwind, declares all CSS custom properties
- `metro.config.js` — wraps config with `withUniwindConfig()`, points at `global.css`
- `src/types/uniwind.d.ts` — **auto-generated** during bundling; provides type completions for className values

All colors are CSS variables, never hardcoded. Use semantic tokens:
- Backgrounds: `bg-bg`, `bg-card`, `bg-bg-secondary`
- Text: `text-text`, `text-text-secondary`, `text-primary`
- Borders: `border-border`, `border-border-strong`
- Semantic: `text-success`, `text-error`, `text-warning`, `text-info`

Use `cn()` from `src/utils/cn.ts` (clsx + tailwind-merge) whenever conditional or merged classes are needed.

Icons use `<Icon name="..." />` from `src/components/ui/Icon.tsx` (Ionicons, wrapped with `withUniwind()` for className color support).

Dark mode is handled automatically by Uniwind — toggle with `Uniwind.setTheme('dark' | 'light')`.

### Shared UI Components

All live in `src/components/ui/`. Key APIs:

| Component | Key props |
|-----------|-----------|
| `Button` | `variant` (primary/secondary/outline/ghost/success/error), `size` (sm/md/lg), `isLoading` |
| `Text` | `variant` (h1–h4/body/body-sm/caption/label), `weight`, `color` |
| `Card` | `variant` (default/primary/subtle), `radius`, `padded` |
| `Input` | `label`, `error`, `hint`, `leftSlot`, `rightSlot`, `size` |
| `StateView` | `loading`, `title`, `description`, `iconName`, `actionLabel` — use for loading/error/empty states |
| `ScreenWrapper` | Wraps screen content with safe area insets |
| `Section` | Titled content block with optional `rightSlot` |

### Path Aliases

`@/*` resolves to the repo root. Use `@/src/...`, `@/assets/...` etc. everywhere.

### Prettier & ESLint

- **Prettier**: print width 140, tab width 4, single quotes, trailing commas (es5), arrow parens omitted for single param
- **ESLint**: flat config (`eslint.config.js`), ESLint 10. Base: `eslint-config-universe/flat/native` + `@tanstack/eslint-plugin-query` (flat/recommended)
- Disabled rules: `prettier/prettier`, `import/order`, `react-native/no-inline-styles`, `import/namespace`, `@tanstack/query/exhaustive-deps`
- Enabled rules: `no-duplicate-imports: error`
- Root JS config files (`*.config.js`) get Node.js globals automatically

### EAS Builds

Configured in `eas.json`. Requires EAS CLI `>= 16.0.0`.

| Profile | Platform | Notes |
|---------|----------|-------|
| `development` | iOS simulator + Android | `developmentClient: true`, internal distribution |
| `preview` | iOS + Android (real device) | internal distribution, no simulator |
| `production` | iOS + Android | `autoIncrement: true`, EAS auto-selects Xcode image |

> Set `extra.eas.projectId` in `app.json` (run `eas init` to populate it automatically).

### CNG (Continuous Native Generation)

This project has **no `ios/` or `android/` directories** — native projects are generated at build time by EAS. Never commit generated native directories. Run `yarn clean` (`expo prebuild --clean`) only when testing locally.
