## Architecture

### Feature-Based Structure

Code is organized under `src/features/<domain>/` with each feature owning its screens, API, hooks, store, and types. Shared infrastructure lives in `src/shared/`.

```
src/
├── features/<domain>/
│   ├── screens/
│   ├── api/           # Axios functions + queryKeys
│   ├── hooks/         # useQuery / useMutation wrappers
│   ├── store/         # Zustand slice (if needed)
│   └── types.ts
├── shared/
│   ├── components/
│   │   ├── app/       # ErrorBoundary, ScreenHeader, ToastProvider, StatusBar
│   │   └── ui/        # Button, Card, Text, Input, Icon, ScreenWrapper, Section, StateView
│   ├── lib/api/
│   │   └── client.ts  # Axios instance + auth interceptor
│   └── utils/
│       └── cn.ts      # clsx + tailwind-merge helper
├── navigation/        # RootNavigator, MainTabsNavigator, types, hooks
├── constants/
│   └── variable.ts    # TAB_BAR_HEIGHT, getTabBarHeight()
└── types/
    └── uniwind.d.ts   # Auto-generated by metro — do not edit
```

### Navigation

Two-level stack/tabs setup:

- `RootNavigator` — native stack, no header, slide-from-right animation
- `MainTabsNavigator` — bottom tabs (Home, Profile), custom height via `getTabBarHeight()`

Navigation types are in `src/navigation/types.ts`. Always use composite screen prop types (`CompositeScreenProps`) for screens nested inside tabs. Use `useRootNavigation()` from `src/navigation/hooks.ts` inside shared components that need navigation.

### API Layer

All API calls go through the shared Axios instance at `src/shared/lib/api/client.ts`:
- Base URL: `https://jsonplaceholder.typicode.com` (replace per project)
- Timeout: 10s
- Auth token automatically injected from AsyncStorage via request interceptor registered once in `useApiClient()` hook

Pattern for a new feature API:
1. Create `src/features/<domain>/api/<domain>.api.ts` — pure functions that accept `axios` as first arg
2. Create `src/features/<domain>/api/queryKeys.ts` — typed const arrays
3. Create hooks in `src/features/<domain>/hooks/` wrapping `useQuery` / `useMutation`; call `useApiClient()` inside

### State Management

- **Server state**: React Query (`@tanstack/react-query`) — default `retry: 1`, `staleTime: 5min`
- **Client/auth state**: Zustand — one store per domain at `src/features/<domain>/store/use<Domain>Store.ts`; export selector hooks (`useCurrentUser`, `useUserActions`) to avoid direct store access in components

### Styling System

Uses **Tailwind CSS v4 + Uniwind** (not NativeWind v4). The setup is:

- `global.css` — imports Tailwind and Uniwind, declares all CSS custom properties
- `metro.config.js` — wraps config with `withUniwindConfig()`, points at `global.css`
- `src/types/uniwind.d.ts` — **auto-generated** during bundling; provides type completions for className values

All colors are CSS variables, never hardcoded. Use semantic tokens:
- Backgrounds: `bg-bg`, `bg-card`, `bg-bg-secondary`
- Text: `text-text`, `text-text-secondary`, `text-primary`
- Borders: `border-border`, `border-border-strong`
- Semantic: `text-success`, `text-error`, `text-warning`, `text-info`

Use `cn()` from `src/shared/utils/cn.ts` (clsx + tailwind-merge) whenever conditional or merged classes are needed.

Icons use `<Icon name="..." />` from `src/shared/components/ui/Icon.tsx` (Ionicons, wrapped with `withUniwind()` for className color support).

Dark mode is handled automatically by Uniwind — toggle with `Uniwind.setTheme('dark' | 'light')`.

### Shared UI Components

All live in `src/shared/components/ui/`. Key APIs:

| Component | Key props |
|-----------|-----------|
| `Button` | `variant` (primary/secondary/outline/ghost/success/error), `size` (sm/md/lg), `isLoading` |
| `Text` | `variant` (h1–h4/body/body-sm/caption/label), `weight`, `color` |
| `Card` | `variant` (default/primary/subtle), `radius`, `padded` |
| `Input` | `label`, `error`, `hint`, `leftSlot`, `rightSlot`, `size` |
| `StateView` | `loading`, `title`, `description`, `iconName`, `actionLabel` — use for loading/error/empty states |
| `ScreenWrapper` | Wraps screen content with safe area insets |
| `Section` | Titled content block with optional `rightSlot` |

### Path Aliases

`@/*` resolves to the repo root. Use `@/src/...`, `@/assets/...` etc. everywhere.

### Prettier & ESLint

- Print width: 140, tab width: 4, single quotes, trailing commas (es5), arrow parens omitted for single param
- ESLint extends `universe` + `universe/native`; `import/order` and inline styles rules are disabled; `no-duplicate-imports` is an error
